วัตถุประสงค์

  • เพื่อพัฒนาระบบควบคุมและอ่านค่าจาก Arduino ผ่านเว็บไซต์ได้แบบเรียลไทม์
  • เพื่อให้ผู้ใช้สามารถทดลอง Sensor ได้โดยไม่ต้องอัปโหลดโค้ดทุกครั้ง
  • เพื่อสร้างสื่อการเรียนรู้ที่เข้าใจง่ายสำหรับผู้เริ่มต้นศึกษาด้าน IoT

แนวคิดและหลักการทำงานของระบบ

ระบบทำงานโดยใช้สถาปัตยกรรมแบบ MQTT (Message Queuing Telemetry Transport) เพื่อสื่อสารระหว่าง Arduino กับเว็บแอปพลิเคชัน

การทำงานหลัก

  • Arduino UNO R4 WiFi เชื่อมต่ออินเทอร์เน็ตผ่าน WiFi และเชื่อมกับ MQTT Broker (mqtt-dashboard.com)
  • บอร์ดจะ Subscribe หัวข้อ miniverse/command เพื่อรอรับคำสั่งจากเว็บไซต์
  • เว็บไซต์จะ Publish คำสั่งไปที่หัวข้อนี้ (เช่น light on, temp, distance)
  • Arduino ประมวลผลคำสั่งและส่งค่ากลับผ่านหัวข้อ miniverse/data เพื่อให้เว็บนำมาแสดงผล
  • ผู้ใช้สามารถดูข้อมูลแบบเรียลไทม์ได้บนหน้าเว็บโดยไม่ต้องอัปโหลดโค้ดใหม่

⚙️ ส่วนประกอบของระบบ
